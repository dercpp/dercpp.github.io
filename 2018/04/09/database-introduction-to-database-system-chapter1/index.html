数据库系统概述

数据库四个基本概念

数据与语义

  • 数据： 数据库中存储的基本对象，描述事务的符号记录
  • 语义： 数据的含义，与数据是不可分的

数据库

  • 定义： 长期存储在计算机内，有组织的、可共享的大量数据的集合
  • 作用：保存数据以供进一步加工处理，抽取有用信息，转换为有价值的知识
  • 基本特征：
    • 数据按一定的数据模型组织、描述与存储
    • 可为各种用户共享、冗余度较小、易扩展
    • 数据独立性高

数据库管理系统

  • 定义： 位于用户应用与操作系统之间的一层数据管理软件
  • 作用： 用于科学的组织、存储数据，高效地维护、获取数据
  • 主要功能：
    • 数据定义功能
      • 提供数据定义语言
      • 定义数据库中地数据对象
    • 数据组织、存储与管理：
      • 分类组织、存储与管理各种数据
      • 确定数据在存储级别上地结构与存取方式
      • 实现数据之间地联系
      • 提供多种存取方法提高存取效率
    • 数据操纵功能
      • 提供数据操纵语言
      • 实现对数据库地基本操作
    • 数据库地事务管理与运行管理
      • 数据地安全性、完整性、多用户对数据地并发使用
      • 发生故障后地系统恢复数据库
    • 数据库地建立与维护功能
      • 提供实用程序/工具，完成数据库数据批量装载，数据库转储，介质故障恢复，数据库地重组织与性能监视等
    • 其他
      • 数据库管理系统与网络中其他软件系统地通信
      • 数据库管理系统之间地数据转换
      • 异构数据库之间地互访与互操作

数据库系统

  • 定义： 在计算机系统中引入数据库后地系统构成
  • 构成：
    • 数据库
    • 数据库管理系统（及其应用开发工具）
    • 应用程序
    • 数据管理员

数据管理技术

  • 数据管理定义： 对数据进行分类、组织、编码、存储、检索、维护

数据库系统特点

  • 数据结构化
    • 数据用数据模型描述
  • 数据的共享性高，冗余度低且易扩充
    • 数据共享的好处：
      • 减少数据冗余，节约存储空间
      • 避免数据之间的不相容性与不一致性
      • 使系统易于扩充
  • 数据独立性高
    • 物理独立性： 用户的应用程序与数据库中数据的物理存储是相互独立的。当数据的物理存储改变了，应用程序不用改变
    • 逻辑独立性： 用户的应用程序与数据库的逻辑结构是相互独立的。数据的逻辑结构变了，应用程序不用改变
    • 数据独立性由数据库管理系统的二级映像功能来保证
  • 数据由数据库管理系统统一管理和控制
    • 数据库管理系统提供的数据控制功能：
      • 数据的安全性保护： 保护数据以防止不合法的使用造成的数据的泄密与破坏
      • 数据的完整性检查： 保证数据的正确性、有效性与相容性
      • 并发控制： 对多用户的并发操作加以控制与协调，防止相互干扰而得到错误的结果
      • 数据库恢复： 将数据库从错误状态恢复到某一已知的正确状态
